# Practical UI 設計規範索引

> 完整設計規範請參考：`docs/Practical-UI.pdf`

## 目錄結構

### 1. Fundamentals (基礎原則) - 頁 14-42

- Minimise usability risks (降低可用性風險) - 頁 15
- Have a logical reason for every design detail (每個設計細節都要有邏輯理由) - 頁 17
- Minimise interaction cost (降低互動成本) - 頁 19
- Minimise cognitive load (降低認知負荷) - 頁 22
- Create a design system (建立設計系統) - 頁 24
- Ensure an interface is accessible (確保介面可訪問性) - 頁 33
- Use common design patterns (使用常見設計模式) - 頁 38
- Use the 80/20 Rule to prioritise (使用 80/20 法則排序優先級) - 頁 40
- Keep costs in mind (考慮成本) - 頁 41
- Be consistent (保持一致性) - 頁 42

### 2. Less is more (極簡主義) - 頁 45-55

- Remove unnecessary information (移除不必要的資訊) - 頁 46
- Remove unnecessary styles (移除不必要的樣式) - 頁 47
- Not all links need to be underlined (不是所有連結都需要底線) - 頁 49
- Use progressive disclosure (使用漸進式揭露) - 頁 50
- Don't confuse minimalism with simplicity (不要混淆極簡主義與簡潔性) - 頁 52
- Design for the smallest screen first (優先為最小螢幕設計) - 頁 54
- Break up choices to speed up decision making (拆解選擇以加速決策) - 頁 55

### 3. Colour (色彩) - 頁 60-102

- Ensure sufficient contrast (確保足夠的對比度) - 頁 61
- Don't rely on colour alone to convey meaning (不要僅靠顏色傳達意義) - 頁 67
- Use system colours to indicate status (使用系統色彩表示狀態) - 頁 69
- Use colour to define a clear visual hierarchy (使用色彩定義清晰的視覺層級) - 頁 71
- Understand light and dark interfaces (理解明暗介面) - 頁 73
- Use black and white for a timeless aesthetic (使用黑白打造永恆美學) - 頁 76
- Add a tinge of colour to black and white (為黑白添加色調) - 頁 78
- Use 1 brand colour (使用單一品牌色) - 頁 79
- Apply the brand colour to interactive elements (將品牌色應用於互動元素) - 頁 81
- Create a monochromatic colour palette (建立單色調色板) - 頁 85
- Use the HSB colour system (使用 HSB 色彩系統) - 頁 87
- Create a light colour palette with rules that govern its usage (建立淺色調色板及其使用規則) - 頁 88
- Create a dark colour palette with rules that govern its usage (建立深色調色板及其使用規則) - 頁 98
- Adjust photo colour temperature to match the colour palette (調整照片色溫以匹配調色板) - 頁 102

### 4. Layout and spacing (佈局與間距) - 頁 104-161

- Group related elements (群組相關元素) - 頁 105
- Create a clear visual hierarchy (建立清晰的視覺層級) - 頁 120
- Test visual hierarchy using The Squint Test (使用瞇眼測試檢驗視覺層級) - 頁 128
- Use depth to create a visual hierarchy (使用深度建立視覺層級) - 頁 129
- Understand the box model (理解盒模型) - 頁 132
- Design @1x using points (使用 points 進行 @1x 設計) - 頁 134
- Create a set of predefined spacing options (建立預定義的間距選項) - 頁 135
- Space elements based on how closely related they are (根據關聯程度調整元素間距) - 頁 137
- Be generous with white space (善用留白) - 頁 144
- Align the main layout to a 12 column grid (主佈局對齊 12 欄網格) - 頁 146
- Align text to improve readability (對齊文字以提升可讀性) - 頁 150
- Try to avoid using multiple alignments (避免使用多種對齊方式) - 頁 153
- Keep related actions close (保持相關動作靠近) - 頁 156
- Make sure important content is visible (確保重要內容可見) - 頁 159
- Ensure your interface is unbreakable (確保介面不會崩壞) - 頁 160
- Use the Rule of Thirds for photos (照片使用三分法則) - 頁 161

### 5. Typography (字體排版) - 頁 163-191

- Use a single sans serif typeface (使用單一無襯線字體) - 頁 164
- Evoke emotion using a second typeface for headings (使用第二種字體為標題增添情感) - 頁 171
- Use regular and bold font weights only (僅使用標準和粗體字重) - 頁 173
- Use a type scale to set font sizes (使用字體比例設定字級) - 頁 175
- Make long body text bigger (增大長文本字級) - 頁 178
- Use at least 1.5 line height for long body text (長文本使用至少 1.5 行高) - 頁 179
- Decrease line height as font size increases (字級增大時減少行高) - 頁 181
- Ensure ideal line length (確保理想的行長) - 頁 182
- Left align text (文字左對齊) - 頁 185
- Decrease letter spacing for large text (大文字減少字距) - 頁 187
- Ensure text on photos is legible (確保照片上的文字清晰可讀) - 頁 188
- Avoid light grey and pure black text (避免淺灰和純黑文字) - 頁 191

### 6. Copywriting (文案撰寫) - 頁 192-211

- Be concise (保持簡潔) - 頁 193
- Use sentence case (使用句首大寫) - 頁 195
- Use plain and simple language (使用簡單明瞭的語言) - 頁 196
- Front-load text (重點前置) - 頁 197
- Use the inverted pyramid (使用倒金字塔結構) - 頁 198
- Limit the use of abbreviations and acronyms (限制縮寫使用) - 頁 200
- Limit the use of UPPERCASE (限制全大寫使用) - 頁 201
- Break up content using descriptive headings and bullets (使用描述性標題和項目符號分段) - 頁 202
- Avoid using "my" on form labels (表單標籤避免使用「我的」) - 頁 204
- Use vocabulary consistently (用詞保持一致) - 頁 205
- Use numerals for numbers (數字使用阿拉伯數字) - 頁 207
- Avoid full stops if possible (盡可能避免句號) - 頁 209
- Ensure text length is similar across similar interface elements (相似介面元素的文字長度應相似) - 頁 210
- Ensure text links describe their destination (確保文字連結描述其目的地) - 頁 211

### 7. Forms (表單) - 頁 213-248

- Stack forms in a single column layout (表單使用單欄堆疊佈局) - 頁 214
- Minimise the number of form fields (最小化表單欄位數量) - 頁 219
- Mark optional fields (標記選填欄位) - 頁 220
- Try to avoid optional fields by using opt-ins (盡量透過選擇性加入避免選填欄位) - 頁 221
- Mark both required and optional fields (同時標記必填和選填欄位) - 頁 222
- Match field width to the intended input (欄位寬度符合預期輸入) - 頁 227
- Stick with conventional form field styles (使用傳統表單欄位樣式) - 頁 229
- Display hints above form fields (在表單欄位上方顯示提示) - 頁 231
- Don't use placeholder text instead of a label (不要用佔位符取代標籤) - 頁 233
- Try to use radio buttons instead of dropdowns (優先使用單選按鈕而非下拉選單) - 頁 235
- Use an autocomplete instead of a long dropdown (使用自動完成取代長下拉選單) - 頁 236
- Use steppers for numeric fields instead of dropdowns (數字欄位使用步進器而非下拉選單) - 頁 238
- Use a checkbox or toggle switch for 2 options (兩個選項使用核取方塊或切換開關) - 頁 240
- Use positive phrasing for checkboxes (核取方塊使用正面措辭) - 頁 242
- Break up long forms into multiple steps (將長表單拆分為多個步驟) - 頁 243
- Group related fields under headings (相關欄位在標題下分組) - 頁 244
- Ensure form field borders are high contrast (確保表單欄位邊框高對比度) - 頁 245
- Write clear error messages (撰寫清晰的錯誤訊息) - 頁 246
- Ensure form field labels are close to their fields (確保表單標籤靠近欄位) - 頁 247
- Validate on submit rather than inline (提交時驗證而非即時驗證) - 頁 248

### 8. Buttons (按鈕) - 頁 250-277

- Define 3 buttons weights (定義 3 種按鈕權重) - 頁 251
- Use a single primary button for the most important action (最重要動作使用單一主要按鈕) - 頁 258
- Use secondary buttons for less important actions (次要動作使用次要按鈕) - 頁 260
- Use tertiary buttons for the least important actions (最不重要動作使用第三級按鈕) - 頁 261
- Try to avoid disabled buttons (避免使用禁用按鈕) - 頁 262
- Left align buttons (按鈕左對齊) - 頁 267
- Ensure button text describes the action (確保按鈕文字描述動作) - 頁 272
- Ensure buttons have a sufficient target size (確保按鈕有足夠的點擊目標大小) - 頁 273
- Balance icon and text pairs (平衡圖示與文字配對) - 頁 275
- Add friction to destructive actions (為破壞性動作增加摩擦) - 頁 277
